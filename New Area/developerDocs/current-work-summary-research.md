### 2.2 애플리케이션 구조 (`평수관리.html`)

**기술 스택**: 순수 HTML5 + CSS3 + Vanilla JS (프레임워크 없음, 단일 파일 완결)

**레이아웃**:
```
<header>  ← 물류센터 평수 관리 시스템 타이틀
<tabs>    ← 층 탭 (2F 전체 / 2F 메자닌 / 4F 전체 / 4-2F 메자닌)
<main>
  ├─ <canvas-area>  ← SVG 도면 렌더링 영역 (스크롤 가능)
  └─ <sidebar>      ← 페인팅 모드 + 회사 목록 + 평수 요약 (290px 고정)
```

### 2.4 구현된 핵심 기능

1. **SVG 도면 렌더링** (`buildFloorSVG`)
   - 스판별 셀을 `<rect>` SVG 요소로 렌더링
   - 각 셀에 평수(平) 텍스트 표시
   - X축 레이블(상단), Y축 레이블(좌측), 스판 치수 레이블(mm→m 변환)
   - 창 크기에 반응하는 스케일링 (`window.resize` 이벤트)

2. **페인팅 모드**
   - **스판 전체 모드**: X열 전체를 한 번에 색칠
   - **개별 셀 모드**: X·Y 교차 단일 셀 선택
   - 마우스 드래그 페인팅 (`mousedown` + `mouseenter`)
   - 우클릭으로 개별 셀/열 지우기

3. **공용면적 제외** (`COMMON_AREAS`)
   - 2F: X1~X2(xi=0), X23~X24(xi=22)의 Y4~Y11 구역
   - 2F: Y10~Y11(yi=9) 행 전체
   - 2-2F: 메자닌 영역이 아닌 셀 전체를 비활성화

4. **메자닌 실측 치수** (`MEZ_XSPAN`, `MEZ_YSPAN_L/R`)
   - 좌측(X6~X12): 실측 X스판 10,200~11,000mm / Y스판 8,450~9,300mm (총 64,250mm)
   - 우측(X15~X24): 실측 X스판 10,400~17,700mm / Y스판 8,500~11,700mm (총 75,300mm)
   - 특정 셀 오버라이드: `MEZ_CELL_PY` (예: X16~X17/Y7~Y8 = 16.3평)

5. **회사 관리** (`companies` 배열)
   - 회사 추가/삭제/이름 편집/색상 변경
   - 최대 15개 색상 팔레트 순환
   - 초기값: 회사 A(#FF6B6B), 회사 B(#4D96FF), 회사 C(#6BCB77)

6. **평수 요약 패널** (`renderSummary`)
   - 현재 층: 회사별 m²·평 합계, 배정 합계, 층 전체 면적
   - 스판별 상세: X축 기준 스판별 회사·면적
   - 전체 층 합계: 모든 층의 회사별 면적 합산

7. **CSV 내보내기** (`exportCSV`)
   - 출력 컬럼: 층, 스판, 회사, 면적(m²), 평수(평)
   - UTF-8 BOM 포함 (`﻿` prefix)
   - 파일명: `물류센터_평수요약.csv`

8. **localStorage 영속성**
   - 키: `warehouse_v3`
   - 저장 내용: `{cellStates, companies, nextId}`

9. **2F 메자닌 점선 박스 표시**
   - 3개 구역: X6~X9(2,069m²), X9~X12(2,069m²), X15~X24(6,223m²)
   - 파란 점선 박스 + 구역명 라벨

10. **야외도크 라벨** (2F, 2-2F의 Y9 행 좌측에 세로 텍스트)

## 3. 핵심 발견사항

1. **완성도 높은 단일 파일 앱**: 별도 빌드 없이 브라우저에서 직접 실행 가능. 기능적으로 완결된 상태.
2. **메자닌 실측 처리**: 도면 실측값(`MEZ_XSPAN`, `MEZ_YSPAN_L/R`)을 별도로 하드코딩하여 표준 격자와 다른 실제 면적을 정확히 계산.
3. **localStorage 버전 키**: `warehouse_v3` — 이전에 `v1`, `v2` 버전이 있었다가 스키마가 변경된 이력 시사.
4. **이미지 자산 다수**: PDF 도면에서 추출된 PNG 이미지들이 참조용으로 존재하나, 현재 HTML에서는 사용하지 않음 (외부 참조용으로 추정).
5. **4-2F 메자닌 구역**: 2-2F와 달리 `MEZ_XSPAN/YSPAN` 실측 적용 없이 단순 격자 계산 사용 중 — 실측 적용이 필요할 수 있음.
6. **메자닌 총면적 불일치**: `renderSummary`에서 `2-2F`의 `totalMm2 = 12052e6` 하드코딩 사용, note의 3,134평(10,361m²)과 다름. 실측값 기준으로 보정한 것으로 보임.

## 4. 제약사항 및 고려사항

- **단일 파일**: 별도 모듈 시스템 없음. 기능 추가 시 파일 내 섹션 관리 필요.
- **localStorage 의존**: 브라우저 새 탭이나 다른 PC에서는 데이터 공유 불가. 데이터 이식성이 필요하면 JSON export/import 기능 고려.
- **반응형 제한**: `canvas-area`는 가로 스크롤 가능하나 모바일 미최적화.
- **SVG 재렌더링 방식**: 도면 전체를 매 페인트마다 DOM 쿼리로 업데이트 (전체 재렌더 없이 부분 업데이트). 성능은 현 규모에서 충분.
- **메자닌 실측치 하드코딩**: `MEZ_XSPAN`, `MEZ_YSPAN_L/R`, `MEZ_CELL_PY` 값이 모두 하드코딩 — 도면 변경 시 코드 수정 필요.
