# 물류센터 평수관리 시스템 개선 - Implementation Plan

> 작성일: 2026-04-24 | 갱신일: 2026-04-30 | 상태: 완료
> Research: developerDocs/current-work-summary-research.md

## 1. 목표

`평수관리.html` 단일 파일 앱의 주요 개선 사항을 단계별로 구현한다.
우선순위는 면적 계산 정확성 → UI 편의 기능 → 데이터 이식성(JSON) → 출력(인쇄/PDF) 순으로 진행한다.

## 2. 구현 범위

**In-scope**
- 2F 바닥 가로×세로 실제 평수 계산 및 표시 (m² ÷ 3.3)
- 램프 등 면적 제외 대상 선택 버튼 생성 (제외 셀 회색 음영 표시)
- 회사별 수동 면적 추가 버튼 (그리드 외 별도 항목)
- 면적 수치 직접 수정 버튼 생성
- 4-2F 메자닌 MEZ 실측 치수 적용
- 스판 셀 커스텀 이름 입력 기능
- JSON import/export 기능
- 도면 없는 층(1층 운영) 회사의 기타 계약면적 직접 입력: 실면적과 계약면적을 각각 독립적으로 직접 입력 가능
- 현황 인쇄/PDF 출력
- 요약 패널: 회사별 층별 분류 표시 + 전체 층 합계
- PPT 다운로드: 층별 슬라이드, 평수 기입, 회사별 요약 표시 (슬라이드 도면이 HTML 화면과 동일한 시각적 표현 — 회사 색상·제외 셀·분할 셀 포함)
- 전체 층별 회사별 요약 화면: 도면 없이 회사와 숫자만 표시하는 전용 화면
- 스판 분할 기능: 1개 스판 안에 2개 회사를 분할 비율로 평수 입력
- 요약 현황 실면적/계약면적 이중 표시: 1F는 실÷77.42%, 2F는 실÷77.55%, 4F는 실÷78.72%, 메자닌은 실면적 동일 표시
- 요약 현황 "기타" 열 추가: 실면적|계약면적 2열, 직접 입력, 계약면적 수식 자동 반영
- 1F 층 추가: 2F와 동일한 그리드(23×10), 자동 면적 계산 없음, 셀 직접 면적 입력

**Out-of-scope**
- 도면 이미지 오버레이 (SVG 배경)
- 공용면적 UI 편집
- 서버/백엔드 연동
- 모바일 최적화

## 3. 작업 단계

### 3.1 단계 1: 실제 평수 계산 검증 및 표시 개선

현재 각 셀의 면적은 스판 치수(mm)를 기반으로 계산되어 있다. 2F 바닥의 가로×세로 실측 치수를 기반으로 m²를 산출하고, `m² ÷ 3.3`으로 평수를 표시함을 명확히 한다.

- [x] `평수관리.html`: 각 셀의 면적 계산 공식 확인 — `(xSpanMm × ySpanMm) / 1_000_000`이 m² 기준인지 검증 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: 평수 변환 공식 통일 — `평 = m² / 3.3` (기존 `3.305785` 등 혼용 여부 확인 후 단일화)
- [x] `평수관리.html`: 요약 패널에서 m²와 평수를 함께 표시하여 계산 근거 투명화
- [x] 2F 전체 면적 합계가 9,530평 (≈31,449m²)에 근접하는지 수동 검증

### 3.2 단계 2: 면적 제거 버튼 (램프 등 제외 구역 선택)

램프·계단 등 임차 면적에서 제외되는 구역을 UI에서 선택하여 면적 합계에서 동적으로 제외할 수 있도록 한다.

- [x] `평수관리.html`: 사이드바에 "면적 제외 모드" 토글 버튼 추가 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: 제외 모드 활성화 시 셀 클릭 → `excluded: true` 상태로 마킹 (빗금/회색 패턴 표시)
- [x] `평수관리.html`: `excluded` 셀은 `renderSummary()` 면적 합산에서 제외
- [x] `평수관리.html`: `excluded` 상태를 `cellStates`에 포함하여 `localStorage` 영속화
- [x] `평수관리.html`: 우클릭 또는 "제외 해제" 버튼으로 제외 취소 기능 추가

### 3.3 단계 3: 면적 수치 직접 수정 버튼

특정 셀의 면적 수치를 스판 치수 계산과 무관하게 수동으로 입력·수정할 수 있도록 한다.

- [x] `평수관리.html`: 셀 더블클릭 또는 "수정 모드" 버튼 활성화 시 인라인 입력 필드 표시 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: 입력값(평 단위)을 `cellStates[key].customArea`로 저장 (m² 역산 포함)
- [x] `평수관리.html`: `customArea`가 있는 셀은 계산된 면적 대신 커스텀 값 우선 사용
- [x] `평수관리.html`: `customArea`를 `cellStates`에 포함하여 `localStorage` 영속화
- [x] `평수관리.html`: 커스텀 값 셀에 시각적 표시 (예: 셀 우상단 아이콘) 추가
- [x] `평수관리.html`: 커스텀 값 초기화 버튼으로 원래 계산값 복원

### 3.4 단계 4: 4-2F 메자닌 실측 치수 적용

현재 4-2F는 단순 격자 계산(X12~X24, Y2~Y8)만 사용. 2-2F와 동일한 방식으로 MEZ 실측 치수를 적용한다.

- [x] 4-2F 도면 확인 후 실측 X스판/Y스판 값 수집 (변경 파일: `평수관리.html` → `MEZ_XSPAN_4F`, `MEZ_YSPAN_4F` 상수 추가)
- [x] `평수관리.html`: `MEZ_XSPAN_4F`, `MEZ_YSPAN_4F` 상수 정의 (X12~X24, Y2~Y8 실측값)
- [x] `평수관리.html`: `calcCellArea()` 또는 면적 계산 로직에서 `4-2F` 분기 추가
- [x] `평수관리.html`: `renderSummary()`의 `4-2F` totalMm2 하드코딩 값을 실측 계산으로 교체
- [x] 2-2F와 4-2F 메자닌 총면적 비교 검증

> **참고**: 실측값이 없으면 사용자에게 확인 후 진행. 2-2F 패턴(MEZ_XSPAN, MEZ_YSPAN_L/R, MEZ_CELL_PY)을 그대로 따름.

### 3.5 단계 5: JSON import/export 기능

localStorage는 브라우저/PC 간 공유 불가. JSON 파일로 내보내기/불러오기 기능을 추가한다.

- [x] `평수관리.html`: 사이드바 하단에 "JSON 내보내기" 버튼 추가 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: `exportJSON()` 함수 — `{cellStates, companies, nextId, version: "v3"}` 구조로 JSON Blob 다운로드
- [x] `평수관리.html`: "JSON 불러오기" 버튼 + 숨김 `<input type="file">` 추가
- [x] `평수관리.html`: `importJSON()` 함수 — 파일 읽기 → JSON 파싱 → version 검증 → `warehouse_v3` localStorage 업데이트 → 페이지 재렌더
- [x] 잘못된 파일 형식 오류 처리 (alert 또는 인라인 메시지)

### 3.6 단계 6: 현황 인쇄/PDF 출력

배정 현황을 인쇄 가능한 형태로 출력한다. CSS `@media print` + `window.print()` 방식 사용.

- [x] `평수관리.html`: "인쇄" 버튼 추가 (헤더 또는 사이드바 하단)
- [x] `평수관리.html`: `printSummary()` 함수 — 현재 층 기준으로 요약 테이블 + SVG 함께 출력
- [x] 브라우저 인쇄 다이얼로그에서 "PDF로 저장" 가능 확인 (별도 라이브러리 불필요)

### 3.7 단계 7: 스판 셀 커스텀 이름 입력 기능

- [x] `평수관리.html`: 요약 패널에 "층별 회사 현황" 섹션 추가 — 각 층(2F, 2-2F, 4F, 4-2F)별로 회사명+면적 표시 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: "전체 합계" 섹션 — 모든 층의 회사별 m²·평 합산 테이블 표시
- [x] `평수관리.html`: 현재 층 선택과 무관하게 전체 층 요약은 항상 표시 (또는 탭/토글로 전환)
- [x] `평수관리.html`: `renderSummary()` 함수를 현재 층 요약 + 전체 층 요약 두 섹션으로 분리
- [x] `평수관리.html`: 인쇄 시 전체 층 요약도 함께 출력되도록 `@media print` CSS 조정

### 3.9 단계 9: 면적 제외 표시 개선 + 수동 면적 추가 버튼

**9-A. 면적 제외 표시 회색 음영으로 변경**
현재 제외 셀은 빗금(hatch) 패턴으로 표시됨. 회색 음영(solid gray fill)으로 변경하여 시각적으로 더 명확히 구분.

- [x] `평수관리.html`: 제외 셀 SVG 스타일 변경 — `hatchPattern` 제거, `fill: #b0b0b0` (또는 `rgba(150,150,150,0.6)`) 적용 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: 기존 `<defs>` hatchPattern 정의 제거 또는 미사용으로 정리

**9-B. 수동 면적 추가 버튼 (회사별 그리드 외 항목)**
그리드 셀과 무관한 별도 면적 항목을 회사에 추가할 수 있도록 한다.
예: "외부 야적장: 300평", "연결 통로: 50평" 등 도면 셀에 포함되지 않는 면적.

- [x] `평수관리.html`: 회사 목록 각 항목 옆에 "＋ 면적 추가" 버튼 추가 (변경 파일: `평수관리.html`)
- [x] `평수관리.html`: 클릭 시 `prompt()`로 항목명과 평수 입력 → `companies[i].extras` 배열에 `{name, py}` 객체 추가
- [x] `평수관리.html`: `renderSummary()` 회사별 합계에 `extras` 합산 포함 (별도 행으로 표시)
- [x] `평수관리.html`: 전체 층 합계 테이블(`renderGlobalSummary()`)에도 extras 반영
- [x] `평수관리.html`: `companies` 배열에 `extras` 포함하여 `localStorage` 영속화
- [x] `평수관리.html`: CSV 내보내기에 extras 항목 별도 행으로 추가 (층="-", 스판="-", 레이블=항목명)
- [x] extras 항목 삭제 버튼 또는 우클릭 삭제

## 4. 기술 결정사항

- **단일 파일 유지**: 모든 변경은 `평수관리.html` 내에서 처리. 외부 의존성 추가 없음.
- **평수 계산 공식**: `평 = m² / 3.3` 통일. `(xMm × yMm) / 1_000_000 / 3.3`.
- **면적 제외**: `cellStates[key].excluded = true` 필드 추가. `excluded` 셀은 요약 합산에서 제외.
- **커스텀 면적**: `cellStates[key].customArea` (평 단위) 필드 추가. 존재 시 계산값 대신 사용.
- **JSON 스키마**: `warehouse_v3` 키 구조 그대로 사용 (`{cellStates, companies, nextId}`). import 시 version 필드로 호환성 확인.
- **인쇄**: CSS `@media print` + `window.print()` 조합. jsPDF 등 외부 라이브러리 사용 금지.
- **4-2F 실측**: 2-2F 패턴(`MEZ_XSPAN`, `MEZ_YSPAN_L/R`) 그대로 확장. 별도 상수명으로 분리.
- **셀 레이블**: `cellMeta[fk][key].label` 문자열 필드. SVG 셀 내 작은 폰트(10px 이하)로 표시.
- **전체 요약**: `renderSummary()` 내부에서 모든 층 데이터를 집계하는 별도 섹션 렌더링. 현재 층 탭 상태와 독립적으로 갱신.
- **제외 셀 표시**: `fill: rgba(150,150,150,0.6)` 단색 음영으로 통일. 빗금(hatch) 패턴 제거.
- **수동 면적 추가**: `companies[i].extras = [{name, py}]` 배열 필드. 요약 합산 및 CSV 내보내기에 포함. localStorage 영속화.
- **PPT 다운로드**: PptxGenJS CDN + `buildFloorSVG(fk, st, true, 1200)` 고정 너비 렌더 → SVG→Canvas→PNG → LAYOUT_WIDE 슬라이드. 파일명 `물류센터_평수현황.pptx`. **HTML 화면 동일 표시 원칙**: 슬라이드 도면은 HTML 도면과 동일한 `buildFloorSVG()` 함수 결과를 그대로 사용 — 회사 색상, 셀 평수 텍스트, 제외 셀 회색 음영(`rgba(150,150,150,0.6)`), 분할 셀(좌/우 2색) 모두 PPT에도 동일하게 반영.
- **요약 전용 화면**: 전체 화면 모달 오버레이. rows=회사, cols=층. `renderSummaryModal()` 별도 함수. 인쇄 지원.
- **스판 분할**: `cellMeta[fk][key].split = {co1Id, co2Id, ratio}`. SVG 좌/우 `<rect>` 2개 렌더. 면적 합산 시 ratio 비례 적용. CSV 두 행 출력.
- **실/계약 이중 표시**: `contractPy(fk, py)` 함수 — `'1F'`: `py/0.7742`, `'2F'`: `py/0.7755`, `'4F'`: `py/0.7872`, 메자닌: 동일. 모달 및 사이드바 테이블 모두 적용. extras는 변환 없이 실면적만 표시.
- **기타 열**: `companies[i].gita = { actualPy, contractPy }`. 실면적 셀 클릭 → `editGita()`, 계약면적 셀 클릭 → `editGitaContract()`. `contractPy`가 존재하면 직접 입력값 사용, 없으면 `actualPy / 0.7755` 자동 계산. 빈 값 입력 시 자동 계산으로 초기화. localStorage 포함. 하위 호환: 기존 `{ actualPy }` 구조 그대로 동작.
- **1F 층**: `FLOORS['1F'].noAutoArea = true`. `getCellAreaMm2()` — noAutoArea 플래그 시 customArea×3.3×1e6 반환, 없으면 0. SVG 셀 기본 면적 텍스트 미표시. `contractPy('1F', py)` = `py/0.7742` (1F 전용 77.42%). FLOOR_ORDER 맨 앞에 배치. COMMON_AREAS['1F'] = 빈 Set.
