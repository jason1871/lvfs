## 1. 요구사항 요약

아암센터(물류센터) 내 입주 브랜드 직원들의 점심·저녁 식사 여부를 관리하는 종이 장부가 있다. 매일 식사 시간마다 직원이 서명(사인)하여 식사 확인을 남기고, 이 종이를 촬영한 이미지를 Claude에게 전달하면 Claude가 서명 유무를 인식하여 일자별로 엑셀 파일에 정리해야 한다. 기존 `아암센터 식수명단_260424.xlsx` 파일이 참조 포맷이며, 이를 기반으로 자동화 프로세스를 설계한다.

## 2. 현황 분석

### 2.1 기존 엑셀 파일 구조 (`아암센터 식수명단_260424.xlsx`)

**시트 구성:**

| 시트명 | 역할 |
|--------|------|
| `Sheet1` | 특정 날짜(2026-04-26) 실제 서명 장부 |
| `양식` | 고정 멤버 등록 양식 (브랜드별 이름 목록) |
| `안내문` | 사용 안내 (내용 비어있음) |

**Sheet1 레이아웃 (2열 병렬 구조):**

```
R2:  일자 | 2026-04-26
R3:  [헤더] NO. | 소속 브랜드 | 이름 | 사인  ||  NO. | 소속 브랜드 | 이름 | 사인
         C2       C3          C4    C5          C7       C8          C9    C10
R4+: 데이터 (브랜드 섹션별 반복)
```

- 좌측(C2~C5)과 우측(C7~C10) 두 열에 병렬로 브랜드 목록 배치
- 브랜드 섹션마다 헤더 행이 반복됨
- 이름이 채워진 행 = 등록 멤버, 빈 행 = 여분 슬롯

**등록 브랜드 및 멤버 현황:**

| 브랜드 | 등록 인원 | 비고 |
|--------|----------|------|
| 시코르 | 5명 확인 (+ 빈 슬롯 ~20개) | 김광섭, 조중훈, 윤동원, 김강윤, 윤선영 |
| 톰보이 | 30+명 슬롯 | 김세종, 강지연, 이준협, 정철민, 김도경 등 |
| BNF | 2명 확인 | 윤상정, (윤경화) |
| 그레이고 | 3명 확인 | 이현간, 박대웅, 정영주 |
| 스타일노리터 | 4명 확인 | 이희숙, 박진혁, 김경인, 고준호 |
| YG | 7명 확인 | 서태현, 송재원, 최중근, 정영미, 홍민찬, 김재경, 이우진 |
| 토박스 | 4명 확인 + 슬롯 | 민덕현, 김제위, 김병직, 김용덕 |
| 운영본부 | 9명 확인 | 노재호, 심종일, 이대희, 김지환, 김시영, 이리나, 김나영, 선우성환, 강민수 |
| JYP | 4명 확인 + 60개 슬롯 | 강보석, 권오령, 박상우, 김준희 외 다수 |

**양식 시트:** 브랜드별 고정 멤버 목록 (Sheet1의 사전 등록 원본)

### 2.2 종이 장부 예상 구조

현재 엑셀 Sheet1이 종이 장부를 그대로 디지털화한 형태이므로, 실물 종이 장부는:
- 상단: **일자** 기재
- 구분: **점심 / 저녁** 표기 (별도 용지 또는 구분선)
- 컬럼: NO. | 소속 브랜드 | 이름 | **사인란** (손서명)
- 사인란에 서명이 있으면 식사 O, 없으면 식사 X

### 2.3 현재 프로세스 (As-Is)

```

수작업으로 엑셀을 업데이트하거나, 종이만 보관하는 것으로 추정.

### 2.4 목표 프로세스 (To-Be)

                                                     │
                                         [엑셀 일자별 자동 기록]
```

## 3. 핵심 발견사항

### 3.1 엑셀 데이터 모델

현재 엑셀은 **날짜 단위 장부** 형식이다. 자동화 시 두 가지 출력 모델이 가능하다:

**모델 A: 날짜별 시트 누적**
```
Sheet: 2026-04-26 점심
Sheet: 2026-04-26 저녁
Sheet: 2026-04-27 점심
...
```
→ 장점: 날짜별 상세 내역 보존  
→ 단점: 시트가 많아져 관리 어려움

**모델 B: 집계 테이블 (권장)**
```
이름 | 브랜드 | 04/26 점 | 04/26 저 | 04/27 점 | 04/27 저 | ... | 합계
김광섭 | 시코르 |    O     |    X     |    O     |    O     | ... |  3
```
→ 장점: 월간 집계, 인원별 현황 한눈에 확인  
→ 단점: 초기 구성 필요

**→ 두 모델 병행 추천**: 날짜별 원본 시트 + 월간 집계 시트

### 3.2 Claude Vision의 서명 인식 방식

Claude Vision은 손글씨 서명이 있는 칸(사인란)을 다음 방식으로 인식:
- **이름 옆 사인란에 필기/서명 존재** → 식사 O
- **사인란이 비어 있음** → 식사 X
- 체크 표시(V, ✓), 이름 재기입, 한자 등 다양한 서명 형태 인식 가능

프롬프트 설계가 중요:
```
"이 식수 명단 이미지에서 각 행의 사인란(마지막 열)에 
서명/필기가 있는지 확인하여 JSON으로 반환하세요.
형식: [{"브랜드": "시코르", "이름": "김광섭", "서명": true/false}]"
```

### 3.3 점심/저녁 구분 방법 (확정)

- **같은 이미지 안에 점심·저녁 섹션이 함께 기재됨**
- 저녁은 매일 발생하지 않으므로 저녁 섹션은 없는 날도 있음
- Claude가 이미지 내 "점심 / 저녁" 표기를 읽어 각각 독립 추출
- 이미지 1장 → 점심 서명 목록 + 저녁 서명 목록(있으면) 동시 반환

### 3.4 기술 구현 방식 (확정: Claude Code 직접 처리 + 토큰 최소화)

**선택 이유**: 별도 Python API 파이프라인 없이 Claude Code 세션에서 이미지를 직접 읽어 처리 → 추가 API 호출 없음, 프롬프트 최소화

**토큰 최소화 전략:**

| 항목 | 전략 |
|------|------|
| Vision 프롬프트 | 자연어 설명 요청 금지 → JSON 직접 반환만 요청 |
| 출력 구조 | `{"lunch": ["이름1", "이름2"], "dinner": ["이름3"]}` 최소 형식 |
| 이미지 크기 | 가능하면 1MB 이하로 압축 촬영 (2000px 이하) |
| 마스터 데이터 | 양식 시트 멤버 목록을 코드에 캐시 → Claude에게 반복 전달 안 함 |
| 교차 검증 | Claude 출력을 마스터 목록과 코드로 대조 → Claude 재질의 없이 처리 |

**처리 흐름:**
```
사용자 → 이미지 첨부 (Claude Code 채팅)
Claude → 이미지 Vision 분석 (단일 호출)
       → JSON 출력: {"date":"2026-04-26","lunch":[...],"dinner":[...]}
Claude → openpyxl로 엑셀 날짜별 시트 생성/업데이트 (Bash Python 실행)
```

## 4. 제약사항 및 고려사항

### 4.1 이미지 품질
- 촬영 각도·조명에 따라 인식률 변동
- 서명이 희미하거나 겹치는 경우 오인식 가능
- **권장**: 평평한 곳에서 위에서 수직 촬영, 충분한 조명

### 4.2 이름 없는 빈 슬롯 처리
- 빈 슬롯(이름 없는 행)에 서명이 있는 경우 → "이름 미등록자" 처리 또는 무시
- 양식 시트의 마스터 목록과 대조 필요

### 4.3 브랜드별 슬롯 수 가변
- JYP 60개 슬롯, 시코르 20개 슬롯 등 브랜드마다 다름
- 이미지 분석 시 빈 슬롯 과다로 처리 속도 영향 가능

### 4.4 월 기준 파일 관리
- 월별로 별도 파일 관리 권장 (`아암센터_식수명단_2026-04.xlsx`)
- 또는 단일 파일에 월간 시트 추가

### 4.5 오인식 검토 단계
- 완전 자동화보다 **Claude 분석 결과를 확인 후 적용**하는 반자동 방식 권장
- Claude가 인식 결과를 텍스트로 먼저 출력 → 확인 → 엑셀 기록

## 5. 결론

**구현 방향**: Claude Code 직접 처리 + openpyxl 엑셀 기록

**확정 스펙:**
- 이미지 1장에 점심·저녁 서명이 함께 기재됨
- Claude Vision이 이미지 내 "점심/저녁" 구분을 직접 읽어 각각 추출
- 출력 형식: 날짜별 시트 (`2026-04-26` 시트명)
- 토큰 최소화: JSON-only 프롬프트, 마스터 목록 코드 캐시

**날짜별 시트 구조 (Output 확정 — A안):**
```
시트명: 2026-04-26

     A         B       C       D
1  브랜드    이름    점심    저녁
2  시코르  김광섭    O       X
3  시코르  조중훈    O       O
4  톰보이  김세종    O       -      (저녁 없는 날은 "-")
...
```

**처리 흐름 (Claude Code 세션):**
```
1. 사용자가 이미지 첨부
2. Claude가 Vision으로 분석 → JSON 추출
   {"date":"2026-04-26",
    "lunch":["김광섭","조중훈",...],
    "dinner":["김광섭",...] or []}
3. 마스터 목록(양식 시트)과 대조하여 O/X 결정
4. openpyxl로 날짜 시트 생성 + 기록
5. 저장 완료 확인 메시지 출력
```

**구현 우선순위:**
1. 마스터 멤버 목록 파싱 (`양식` 시트 → 브랜드:이름 딕셔너리)
2. Vision JSON 추출 프롬프트 최적화
3. 날짜별 시트 생성 + O/X 기록 (openpyxl)
4. 저녁 없는 날 `-` 처리 로직
