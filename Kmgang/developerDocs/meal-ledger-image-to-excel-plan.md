# 물류센터 식수 명단 이미지 → 엑셀 정리 자동화 - Implementation Plan

> 작성일: 2026-04-26 | 갱신일: 2026-05-07 | 상태: Completed
> Research: developerDocs/meal-ledger-image-to-excel-research.md

## 1. 목표

사용자가 식수 명단 종이 장부를 촬영한 이미지를 Claude Code 채팅에 첨부하면, Claude가 Vision으로 이미지를 분석하여 점심·저녁 서명자를 추출하고, `아암센터 식수명단.xlsx`에 날짜별 시트 및 월별 집계 시트를 자동 생성·기록한다. 월 마감 시 해당 월 데이터를 별도 파일로 저장한다.

---

## 2. 구현 범위

**In-scope:**
- 이미지에서 점심·저녁 서명자 이름 추출 (Claude Vision)
- `양식` 시트 마스터 멤버 목록 파싱
- 날짜별 시트 생성 (`YYYY-MM-DD` 시트명)
- 월별 집계 시트 생성 (`YYYY-MM_집계` 시트명) — 멤버별 점심·저녁 누적 횟수 자동 갱신
- 월 마감 기능 — `--close-month YYYY-MM` 인수로 해당 월 데이터를 별도 파일 저장
- 컬럼 구조: `브랜드 | 이름 | 점심 | 저녁` (O / X / 빈칸)
- 저녁 없는 날 빈칸 처리 (저녁 섹션 자체가 없는 날은 저녁 열 기입 안 함)
- 이름과 사인이 모두 명확히 기입된 행만 식사 인원으로 집계 (둘 중 하나라도 비어있으면 제외)
- 인식 불가 이름 → 노랑 셀 표시 (사용자 수동 확인 유도)
- 마스터에 없는 이름이어도 서명 있으면 포함 — 이름 인식 불명확 시 노랑 셀 표시
- **이름 중복 감지 시 노랑 셀 표시** (같은 날 동일 이름이 중식 또는 석식 목록에 2회 이상 등장하면 황색 하이라이트)
- 마스터 미등록 신규 인원은 사용자가 직접 등록하지 않는 한 마스터 등록 문의 없이 노랑 표시로만 처리
- 중복 실행 시 기존 시트 덮어쓰기

**Out-of-scope:**
- 이미지 전처리 (압축·크롭 자동화)

## 3. 작업 단계

### 3.1 단계 1: 마스터 멤버 목록 파싱 스크립트

- [x] `parse_master.py` 작성 (변경 파일: 신규 생성)
  - `openpyxl`로 `아암센터 식수명단_260424.xlsx`의 `양식` 시트 로드
  - 브랜드·이름 추출 → `[{"brand": "시코르", "name": "김광섭"}, ...]` 딕셔너리 생성
  - 이름 없는 빈 슬롯 제외
  - 결과를 `master_members.json`으로 저장

**양식 시트 파싱 규칙** (분석 결과):
```
C2: 브랜드, C3: 이름 (좌측)
C5: 브랜드, C6: 이름 (우측)
이름이 비어있으면 skip
```

### 3.2 단계 2: Vision 분석 프롬프트 설계

- [x] Vision 프롬프트 최적화 (변경 파일: `process_meal.py` 내 상수)
  - 토큰 최소화: JSON만 반환, 설명 금지
  - 프롬프트:
    ```
    이 식수 명단 이미지에서 서명(사인)이 있는 사람의 이름만 추출하세요.
    점심과 저녁 섹션을 구분하여 JSON으로만 응답하세요. 설명 없이.
    형식: {"date":"YYYY-MM-DD","lunch":["이름1","이름2"],"dinner":["이름3"]}
    저녁 섹션이 없으면 "dinner":[]
    ```
  - 날짜는 이미지에서 읽거나 없으면 오늘 날짜 사용

### 3.3 단계 3: 메인 처리 스크립트

- [x] `process_meal.py` 작성 (변경 파일: 신규 생성)
  - 입력: Vision JSON (커맨드라인 인수)
  - `master_members.json` 로드
  - 이름 매칭 규칙:
    - 마스터에 있는 이름 → 정상 처리
    - 마스터에 없는 이름이어도 서명 있으면 포함 (노랑 셀 표시)
    - 이름 인식 불명확 (빈 문자열·특수문자) → 노랑 셀 표시
  - `openpyxl`로 `아암센터 식수명단.xlsx` 열기
  - 날짜 시트 없으면 생성, 있으면 덮어쓰기
  - A열: 브랜드, B열: 이름, C열: 점심(O/X), D열: 저녁(O/X/빈칸)
  - 월별 집계 시트 자동 갱신
  - 저장

### 3.4 단계 4: 날짜별 시트 생성 로직

- [x] 시트 생성 함수 구현 (변경 파일: `process_meal.py`)
  - 시트명: `YYYY-MM-DD` (예: `2026-04-26`)
  - 헤더 행: `브랜드 | 이름 | 점심 | 저녁`
  - 마스터 멤버 순서 그대로 기록 (브랜드 그룹 유지)
  - 점심 서명자 → C열 `O`, 미서명 → `X`
  - 저녁 서명자 → D열 `O`, 미서명 → `X`, **저녁 섹션 없는 날 → D열 빈칸 (기입 안 함)**
  - 마스터 외 이름 / 인식 불명확 이름 → 이름 셀 노랑(#FFFF00) 배경
  - 헤더 행 굵게, 배경색 적용 (가독성)

### 3.5 단계 5: 월별 집계 시트 생성 로직

### 3.6 단계 6: 월 마감 기능

  - 출력 파일명: `YYYY-MM_아암센터_식수명단.xlsx`

## 4. 기술 결정사항

| 항목 | 결정 | 이유 |
|------|------|------|
| 이미지 분석 | Claude Code Vision (직접 처리) | 별도 API 호출 없음, 토큰 절약 |
| 엑셀 라이브러리 | `openpyxl` | 순수 Python, COM 불필요 |
| 멤버 목록 캐시 | `master_members.json` | Claude에 반복 전달 안 함 |
| 이름 매칭 | exact match 우선, 불일치·불명확 시 노랑 표시 | 오인식 이름은 사용자가 확인 |
| 마스터 외 이름 | 서명 있으면 포함, 노랑 표시 | 누락 방지 + 사용자 확인 유도 |
| 저녁 없는 날 | `dinner:[]` → D열 빈칸 (기입 안 함) | 사용자 요청 반영 |
| 월별 집계 | `YYYY-MM_집계` 시트 자동 갱신 | 사용자 요청 반영 |
| 월 마감 | `--close-month` 인수로 별도 파일 저장 | 사용자 요청 반영 |
| 시트 중복 | 덮어쓰기 | 재처리 허용 |
| Python 버전 | 3.8+ | openpyxl 호환 |

**필요 패키지:**
```bash
pip install openpyxl anthropic
```
